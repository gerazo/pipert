Detailed Description

template<class T>
class pipert::Receiver< T >

Receiver side object of the pipeline over the network.

Receiver allocates it's own thread where it keeps polling and reading data from the given socket. Socket handling functions come from the constructor parameter named connection. See UDPConnection class for details.

Receiver is handled automatically from the Scheduler (see SchedulerImpl::Start() and SchedulerImpl::Stop()). To use a Receiver in your pipeline you just need to create the object with the proper parameters.

Template Parameters
TType of the Channel where the received data goes.
